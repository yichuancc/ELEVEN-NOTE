# Git相关

## Git出现“filename too long”错误处理

Windows系统下，在Git使用过程中，出现”filename too long”错误提示。直译成中文的意思就是：”文件名过长“。导致始终无法进行将文件添加到Git树中。因为.NET项目的层次比较深而且文件夹和文件名称也比较长。GIt使用了旧版本的Windows API，导致它限制文件名不能超过260个字符。所以Git的Windows客户端默认是禁用长名称支持的。开启长名称支持即可。

打开PowerShell或者Cmd，在命令行窗口中运行：继续进行Git操作就可以成功了。

```shell
git config --system core.longpaths true
```

## Idea切换分支提示

使用 Git 进行版本管理时,当我们切换分支时会出现如上的提示，出现上述界面的原因是我们在当前分支的代码没有 commit,当我们需要切换到其它分支的时候,Git 就会提示我们要进行哪些操作

下面有三个选项 Smart checkout、Force checkout、don`t checkout

![image-20221128150354198](images/image-20221128150354198.png)


1. `Smart checkout`：当前分支未提交的代码会被带到你要切换的那个分支上去
2. `Force Checkout`：就不会把冲突的这部分内容带到目的分支,但是你在当前分支修改的所有内容就会被删除,就算你再切回来也找不到了,所以需要慎重哦。
3. **don`t checkout**：不进行分支切换,还是留在当前分支


## 命令参数解释

| t    | 显示档案文件列表内容（已压缩的文件）    |
| ---- | --------------------------------------- |
| x    | 解压档案文件                            |
| c    | 生成档案文件，创建打包文件              |
| f    | 指定档案文件名称，f必须放在命令参数最后 |

# nginx正向代理

## https请求

参考链接：https://blog.csdn.net/wangshanggang/article/details/143499660

说明：nginx需要增加了ngx_http_proxy_connect_module模块的，已经编译好的windows日 exe文件，可以正向代理https请求。

来源地址：https://github.com/dyq94310/nginx-build-msys2/releases
使用方法：直接替换本地nginx.exe文件启动即可

增加配置：

```json
	server {
		listen 10010;
		resolver 8.8.8.8 114.114.114.114 valid=30s;
		
		# 启用 CONNECT 方法支持（HTTPS 关键配置）
		proxy_connect;
		proxy_connect_allow all;
		proxy_connect_connect_timeout 10s;
		proxy_connect_read_timeout 10s;
		proxy_connect_send_timeout 10s;

		# HTTP 代理
		location / {
			proxy_pass http://$http_host$request_uri;
			proxy_set_header Host $http_host;
		}

		# HTTPS 代理特殊处理
		location ~* ^/(.*) {
			# 对于 CONNECT 请求（HTTPS），不需要 proxy_pass
			# 模块会自动处理隧道建立
		}
	}
```

内网电脑代理外网10010端口即可

# 软链接方式清理C盘

wiztree 工具查看占用的大小

```powershell
# 1. 完全关闭 Qoder

# 2. 创建目标目录
New-Item -ItemType Directory -Path "D:\QoderData" -Force

# 3. 移动目录
Move-Item "C:\Users\cc\.qoder" "D:\QoderData\.qoder"

# 4. 创建符号链接
New-Item -ItemType SymbolicLink -Path "C:\Users\cc\.qoder" -Target "D:\QoderData\.qoder"

验证

Get-Item "C:\Users\cc\.qoder" | Select-Object FullName, LinkType, Target

看到 LinkType: SymbolicLink 就说明成功了，然后重新打开 Qoder 正常使用。
```
